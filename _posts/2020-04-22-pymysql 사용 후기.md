### <span style="color:#F7DC6F;"> 통계학도의 pymysql 사용기  </span>
--------------

최근 진행된 프로젝트에서 $\textsf{python}$으로 작업을 했습니다. 주로 $\textsf{R}$을 사용했던지라 $\textsf{python}$을 집중적으로 다루어본 것은 처음이었습니다. 이번 프로젝트는 DB에 직접 데이터 프레임을 올리는 작업까지 포함하고 있어서 DB를 다룰 기회가 없었던 저에게는 정말 좋은 기회였습니다. 

우선 이번 프로젝트를 통해 느낀 것이 있습니다.

- DB는 어렵다.

- 데이터 전처리에는 확실히 $\textsf{R}$이 $\textsf{python}$보다 편하다.

먼저, DB에 대해서 말해보겠습니다. 

저는 DB에 대해서는 거의 무지한 상태였습니다. 그래서 DB에 접속하는 법이라든지 테이블을 보는 방법도 몰랐습니다. $\textsf{HeidiSQL}$ 프로그램(무료)을 받아서 DB에 접속도 해보고 DB 구성을 확인할 수 있었습니다.

$\textsf{HeidiSQL}$을 통해 DB에 어떤 테이블이 있는지 확인을 했으면 테이블을 전처리 작업을 해야하는 $\textsf{python}$ 환경으로 불러와야합니다. $\textsf{pandas}$의 함수 $\textsf{`pd.read\_sql(query ,conn)'}$을 통해 테이블을 $\textsf{pandas DataFrame}$ 객체로 바로 받을 수 있습니다. 하지만 만약 내가 받아야할 테이블이 너무 커서 DB의 쿼리 유지시간을 초과해서 받아야하는 경우 지긋지긋한 '( 2013 - Lost Connection to MySQL server during query )'가 발생할 가능성이 있습니다. 근본적인 해결방법은 DB의 'wait_timeout' 옵션을 늘리는 방법(Stack Overflow에서 본 해결책)이 있습니다. 하지만 저는 당장 받아서 써야하는 상황이라 다음과 같이 해결했습니다.
```python
conn = pymysql.connect(host='xxx.xx.xx.xxx', user='id', password='password',
                        db='db_name', charset='utf8') 
                        
df = pd.DataFrame()

sql_query1 = "select * from table LIMIT 10000000"
for chunks in pd.read_sql(sql_query1, conn, chunksize=100000):
    df = df.append(chunks)

sql_query2 = "select * from df LIMIT 20000000 OFFSET 10000000"
for chunks in pd.read_sql(sql_query2, conn, chunksize=100000):
    df = df.append(chunks)
```
내려받을 때 chunksize 옵션을 통해 10만개 행을 가지는 batch로 두개의 query로 나누어서 데이터를 추가했습니다. query에 LIMIT은 내려받을 행의 수를 의미하고 OFFSET은 무시할 행의 수를 의미합니다. "select * from df LIMIT 20000000 OFFSET 10000000"은 처음 1000만개는 무시하고 2000만번째 행까지 내려받는 쿼리입니다. 혹시 $\textsf{`pd.read\_sql(query ,conn)'}$ 중 '( 2013 - Lost Connection to MySQL server during query )'가 발생한다면 위와 같이 해보시길 바랍니다. 

테이블을 내려받는 것은 테이블을 Insert 것에 비하면 매우 수월합니다. 올리는 것 역시 $\textsf{DataFrame}$을 DB에 바로 올릴 수 있는 매우 편리한 $\textsf{pd.to\_sql(table\_name, conn)}$ 메소드가 존재합니다.  하지만 처음 DB에 올리는 것이라면 대부분 에러가 발생할 것이라고 생각합니다. 저는 테스트를 위해 단 두개의 행을 가지는 $\textsf{DataFrame}$을 올리는 데는 $\textsf{`pd.to\_sql(table\_name, conn)'}$을 통해 성공했습니다. 하지만 최종적으로 생성된 행이 100만개정도인 $\textsf{DataFrame}$을 올리는데는 실패했습니다. 계속 도전했지만 실패해서 DB에 테이블을 먼저 생성하고 레코즈들을 넣는 방식으로 진행했습니다. 역시나 이 방법도 순탄하지 않았습니다. 그 이유는 다양하겠지만 제가 겪은 에러와 그 원인은 다음과 같습니다.

에러

- InternalError: (1054, "Unknown column 'inf' in 'field list'")
- DataError: (1406, "Data too long for column 'column_a' at row ###")

원인

- $\textsf{pandas DataFrame}$이 nan, inf나 -inf 값을 가지고 있는 경우
- DB가 $\textsf{pandas DataFrame}$의 dtype을 인식 못하는 경우
- 내가 생성한 table의 column에 허용되는 길이를 변수의 길이가 초과할 경우

위의 원인 중 처음 두개는 미리 전처리 작업을 통해 해결해야 합니다.
```python
df['date'] = df['date'].map(str)
df = df.replace([np.inf, -np.inf], np.nan)
df = df.where((pd.notnull(df)), None)
```
'datetime'도 에러가 발생할 가능성이 있으니 그냥 'string'으로 변환하는 것도 나쁘지 않은 것 같습니다. 그리고 inf, -inf는 nan으로 바꾼 후, 'None'으로 변경하면 됩니다([Stack Overflow](https://stackoverflow.com/questions/14162723/replacing-pandas-or-numpy-nan-with-a-none-to-use-with-mysqldb)).


이제 DB에 테이블 생성을 해보겠습니다. 구글링을 해서 $\textsf{pymysql}$ 테이블 생성 예제를 보면 열이 2~3개인 경우처럼 매우 단순합니다. 근데 만약에 실제로 생성한 $\textsf{DataFrame}$의 열이 100개가 넘어가면 일일히 변수명과 변수 타입을 지정해주는 것은 매우 번거롭고 시간이 많이 걸리는 작업입니다. 우선 $\textsf{DataFrame}$의 dtype(object, float64, int)들을 DB table 생성시에 정의할 dtype(VARCHAR, FLOAT, INT 등)으로 변경하는 작업이 필요했습니다.   
```python
convert_dtype= {"object":"VARCHAR(20)", "float64":"FLOAT", "int64":"INT"}

df_column = {a:b for a, b in zip(df.columns, df.dtypes)}

table_dtype = {k:convert_dtype.get(str(df_column.get(k))) for k in df_column.keys()}
table_dtype['column_a'] = 'VARCHAR(30)'
```
$\textsf{python}$ 기본기가 매우 부족해서 위와 같은 방식으로 했으나 익숙하시다면 효율적인 방법으로 하시면 될 것 같습니다. 이 코드를 짜다가 $\textsf{python}$에 'dictionary' 자료형이 유용하다고 느꼈습니다. 저는 $\textsf{R}$만 사용하다보니 'dictionary'의 사용법이 매우 어렵고 익숙치 않았습니다. 위의 에러에서 'column_a'의 변수의 길이가 너무 커서 발생한 문제이니 넉넉하게 길이 30의 'VARCHAR(30)'을 따로 주었습니다.

위와 같이 table의 dtype을 정의했으면 다음의 코드로 table을 생성하면 됩니다.
```python
conn = pymysql.connect(host='xxx.xx.xx.xxx', user='id', password='password',
                        db='db_name', charset='utf8') 

cur = conn.cursor()
create_query = "create table new_table ( "+ ','.join([x + " " + y for x, y in table_dtype.items()]) + " ) ENGINE=InnoDB DEFAULT CHARSET=utf8 "

cur.execute(create_query)

conn.commit()
```
위와 같이 할 경우,  db_name이라는 DB에 new_table 생성된 것을 확인할 수 있습니다. 여기서 query에 $\textsf{CHARSET=utf8}$ 옵션을 넣지않으면 한글 데이터 입력시에 에러가 발생할 수 있습니다. 

테이블을 생성했다면 이제 레코즈들을 넣어야합니다. 그 전에 $\textsf{DataFrame}$의 행을 하나의 tuple로 갖는 list로 변환해야 합니다. 그 다음에 $\textsf{pymysql}$의 $\textsf{pymysql}$
```python
data = [tuple(x) for x in df.values]

try:
    with conn.cursor() as curs:
        insert_query = "insert into new_table (" + ', '.join(df.columns) + ") values (" + ','.join(['%s' for x in range(df.shape[1])]) + ")"
        curs.executemany(insert_query, data)
        
    conn.commit()

finally:
    conn.close()
```

 $\textsf{R}$이 확실히 익숙한 언어라서 그런 것일 수 있지만 확실히 $\textsf{dplyr}$ 패키지는 $\textsf{pandas}$보다는 매우 직관적이고 강력한 데이터 전처리 패키지라고 생각했습니다.
